# KineTwin — รายละเอียดโปรแกรม (Project Description)

> เอกสารสำหรับส่งคณะกรรมการจัดงาน · Digital Aiding 4 Aging Hackathon 2026 (AI Vibe Coding)

---

## 1. ชื่อโครงการ
**KineTwin (Kinematic Digital Twin)** — ระบบตรวจจับและวิเคราะห์การเคลื่อนไหวมือเพื่อ
**คัดกรองภาวะ Learned Non-Use** ในผู้สูงอายุ ผ่าน **วิดีโอธรรมดาจากกล้องมือถือ/เว็บแคม**

---

## 2. บทสรุป (Executive Summary)
KineTwin ให้ผู้ใช้ **ถ่ายคลิปวิดีโอมือยกแก้ว** (ภารกิจ Reach-Grasp-Lift ซึ่งเป็นกิจกรรม
ประจำวันจริง = ADL) แล้วระบบ **ตรวจจับมือด้วย AI (MediaPipe Hands)** หา 21 จุดสำคัญ
บนมือทุกเฟรม จากนั้น **คำนวณเชิงจลนศาสตร์ (kinematics)** ออกมาเป็น 3 คะแนนตามที่โจทย์
กำหนด — **Speed, Accuracy, Quality of movement** — และ **เทียบกับ "แฝดดิจิทัล" (Digital
Twin) ที่เป็นค่าปกติ** เพื่อทำนายว่าผู้ใช้ถนัดมือข้างไหน และ **flag มือข้างที่มีแนวโน้ม
"Learned Non-Use"** (มือที่ผู้ป่วยเลิกใช้จน "ลืม" การทำงาน)

จุดเด่นคือการแสดงผลแบบ **split-screen เรียลไทม์**: ฝั่งซ้ายคือวิดีโอผู้ป่วยพร้อม
**โครงมือ 21 จุด overlay** ทับบนภาพ, ฝั่งขวาคือ **skeleton อ้างอิงค่าปกติ** เล่นซิงค์กัน
→ กรรมการ/แพทย์/ผู้ป่วย **เห็นทันทีว่ามือข้างไหนขยับช้า/ไม่คล่องกว่า** และมีตัวเลข
สรุปผลปรากฏบนหน้าจอ

---

## 3. ปัญหาและแรงบันดาลใจ
- **Learned Non-Use** เป็นภาวะที่ผู้ป่วย (หลัง stroke / สูงวัย) **เลิกใช้แขนข้างที่อ่อนแรง**
  จนสมอง "เรียนรู้" ที่จะไม่ใช้ → ยิ่งนานฟื้นฟูยากขึ้น
- การประเมินในคลินิกต้องใช้ **ผู้เชี่ยวชาญ + เครื่องมือพิเศษ** → คัดกรองได้น้อยครั้ง
  และไม่ทั่วถึง
- จึงขาด **เครื่องมือคัดกรองเบื้องต้นที่เข้าถึงง่าย** (ใช้จากมือถือ/เว็บ ไม่ต้องเซ็นเซอร์ติดตัว)

---

## 4. วัตถุประสงค์
1. ตรวจจับการเคลื่อนไหวมือจาก **วิดีโอ RGB ธรรมดา** (ไม่ต้อง depth sensor)
2. คำนวณ **Speed / Accuracy / Quality** ตามนิยามในโจทย์
3. ทำนาย **ความถนัด (handedness)** และ **flag Learned Non-Use** ข้างที่ผิดสังเกต
4. แสดงผลให้ **เข้าใจง่าย เห็นภาพเทียบเคียง** เพื่อใช้คัดกรอง/ติดตามการฟื้นฟู

---

## 5. ความสามารถของโปรแกรม (Features)
| ฟีเจอร์ | รายละเอียด |
|---|---|
| **อัปโหลดวิดีโอ** | รับไฟล์วิดีโอมือทำภารกิจยกแก้ว |
| **Hand Landmark Detection** | MediaPipe Hands หา **21 จุด** บนมือทุกเฟรม (เรียลไทม์บนการเล่นย้อน) |
| **Split-screen เปรียบเทียบ** | ซ้าย = ผู้ป่วย (พร้อมโครงมือ overlay สีฟ้า) · ขวา = skeleton อ้างอิงค่าปกติ · เล่นซิงค์กัน ปรับขนาดพอดีจอ |
| **คะแนน 3 ตัว** | Speed / Accuracy / Quality (0–100) + ค่าผสม (composite) |
| **เทียบกับค่าปกติ** | แสดง % ความเร็ว/ความลื่นไหล เทียบกับ Digital Twin |
| **Learned Non-Use flag** | บอกมือข้างที่น่าสงสัย "ลืมใช้" + ระดับความรุนแรง (Normal/Mild/Significant) |

## 6. วิธีการทำงาน (Pipeline / Architecture)

```
[วิดีโอมือยกแก้ว]
        │
        ▼  (1) ตรวจจับมือ
   MediaPipe Hands  →  21 จุด/เฟรม (world 3D + image 2D)
        │
        ▼  (2) คำนวณเชิงจลนศาสตร์ (pure math, ไม่ใช่ ML)
   • Speed    = เส้นทางข้อมือ / เวลา
   • Accuracy = ความตรงของเส้นทาง (net/path)
   • Quality  = dimensionless jerk (ความลื่นไหล, ค่าต่ำ = ขยับกระตุก)
        │
        ▼  (3) เทียบกับ Digital Twin (ค่าปกติ)
   ปรับเป็นคะแนน 0–100 + ทำนายความถนัด + flag Learned Non-Use
        │
        ▼  (4) แสดงผล
   Split-screen (วิดีโอ+overlay ⟂ skeleton อ้างอิง) + แผงคะแนน
```

**สถาปัตยกรรมซอฟต์แวร์ (แยกชัด เป็นเลเยอร์):**
- **AI layer** (`src/ai`) — MediaPipe tracking + ฟังก์ชันคำนวณ metric (pure Python) + pipeline
- **Backend** (`src/backend`) — FastAPI, stateless endpoint `POST /analyze-one`
  (รับวิดีโอ → ตรวจจับ → คำนวณ → คืนคะแนน + landmark ทุกเฟรมสำหรับ overlay)
- **Frontend** (`src/frontend`) — React + Vite + TypeScript, split-screen + canvas overlay +
  แผงคะแนน
- **Reference** — skeleton จลนศาสตร์ 21 จุด (Kinematic Digital Twin) สร้างจาก Blender,
  export เป็น trajectory JSON ใช้เป็น baseline "ค่าปกติ"

> ออกแบบ **stateless** (ไม่มี DB / login / session) — คำขอเข้ามา วิเคราะห์ คืนผล จบ

## 7. นวัตกรรมและจุดเด่น
1. **Kinematic Digital Twin** — ตัวอ้างอิง "ค่าปกติ" เป็นโครงกระดูกจลนศาสตร์ 21 จุด
   **ชุดเดียวกับที่ใช้วิเคราะห์** → เปรียบเทียบได้รัดกุมทางคณิตศาสตร์ ไม่ใช่ภาพประกอบ
2. **AI น้อยที่สุด โปร่งใส** — ใช้ MediaPipe (โมเดลสำเร็จติดเครื่อง) เป็น "ตา" ตัวเดียว;
   ส่วน 3 คะแนนเป็น **สูตรจลนศาสตร์ล้วน** ตรวจสอบ/อธิบายได้ (เหมาะกับการแพทย์)
3. **Visual เปรียบเทียบเรียลไทม์** — overlay โครงมือทับวิดีโอ + split-screen กับค่าปกติ
   → กรรมการเข้าใจผลใน 3 วินาที
4. **ใช้ได้จากเว็บ** — ไม่ต้องติดตั้ง ไม่ต้องเซ็นเซอร์พิเศษ

## 8. เทคโนโลยีที่ใช้ (Tech Stack)
| ชั้น | เทคโนโลยี |
|---|---|
| Hand detection | **Google MediaPipe Hands** (on-device, ไม่ใช้ cloud AI) |

## 9. วิธีใช้งาน (User Flow)
1. เปิดเว็บแอป (localhost หรือ URL)
2. **ขั้นที่ 1** — เลือก/อัปโหลดวิดีโอมือยกแก้ว
3. **ขั้นที่ 2** — ดู **split-screen เปรียบเทียบ**: ซ้าย = ผู้ป่วย (มีโครงมือ overlay) · ขวา =
   skeleton อ้างอิง · กด Play เล่นพร้อมกัน
4. **ขั้นที่ 3 — Analyze movement** — กดปุ่ม → ระบบคำนวณ → แสดง
   - คะแนน Speed / Accuracy / Quality (0–100)
   - % ความเร็วและความลื่นไหล เทียบค่าปกติ
   - คำตัดสิน: Normal / Mild impairment / **Significant impairment (suspected Learned Non-Use)**

---

## 10. ประโยชน์และการขยายผล
- **คัดกรอบเบื้องต้น** ที่บ้าน/ชุมชน → เข้าถึงการฟื้นฟูเร็วขึ้น
- **ติดตามการฟื้นฟู** ระหว่างนัด (วัดว่าคะแนนดีขึ้นไหม)
- สอบเทียบกับมาตรฐานทางคลินิกในอนาคตได้
- ขยายเป็น live webcam (real-time) และภารกิจ ADL อื่นๆ ได้

---

## 11. ข้อจำกัดปัจจุบัน (Transparent)
- ตัวอ้างอิง (Digital Twin) เป็น **kinematic skeleton** (โมเดลมือสมจริงกว่านี้สามารถสลับเข้าได้
  โดยไม่ต้องแก้ metric/API)
- การวัด Speed เป็น relative-scale → ใช้ได้ดีที่สุดเมื่อถ่ายจากระยะ/มุมใกล้เคียงกัน
- ปัจจุบันรับ **วิดีโออัปโหลด** (โหมด live camera สามารถเพิ่มได้)
